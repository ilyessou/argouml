Access the source code repository for this project in one of the following ways:

  • Browse source code on-line to view this project's directory structure and files.


  • Download source code archives, if any are available, to copy read-only source files for this project on to your local system.


  • You can use CVS to check out your own copies of project source code files. CVS is an open source version control for keeping track of all modifications to project source code files. You can choose either a command-line CVS client or WinCvs with a Windows interface. What you can do with source files depends upon your role in this project (Roles?). Anyone can access the files for reading using the guest account (with password "guest").

CVS Repository contents

ArgoUML has a few CVS directories and modules. Their purpose is to simplify the use of cvs by allowing you to check out only the necessary part of ArgoUML and not everything. Below is a list of all modules and instructions on what module is required to do what.

  • argouml: everything under argouml.
  • argouml/src_new: contains the src of the current development
  • argouml/tools: contains tools that constitute most of the development environment for ArgoUML.
  • argouml/lib: contains the libs used by ArgoUML.
  • argouml/tests: contains the JUnit tests for everything in src_new.
  • argouml/modules: contains auxiliary modules. ArgoUML can work without them but they add extra features.
  • argouml/documentation: contains the documentation source for the manual, quick-guide and cookbook.
  • argouml/src: contains the old (no longer used) sources. Currently empty.
  • argouml/www: contains the web site source
  • gef: everything under gef.
  • gef/src: the source of the gef library.
  • gef/lib: the libraries needed to build gef.

Required for building ArgoUML:

  • argouml/src_new
  • argouml/lib
  • argouml/tools

Required for creating documentation:

  • argouml/documentation
  • argouml/lib
  • argouml/tools
  • argouml/www (needed to install the documentation on the web site)
  • (docbook-xsl and fop must be downloaded separately).

CVS command line checkout instructions

(Skip to WinCvs checkout instructions)

Before you can do a cvs checkout, you must login to that specific cvsroot.

  1. The command to login is:

    cvs -d :pserver:username@cvs.tigris.org:/cvs login
    Login with your own password, the same as the one you use to access the site. If you access the source in read-only mode your username is "guest" and your password is "guest".

    Then you can check out everything with:

    cvs -d :pserver:username@cvs.tigris.org:/cvs checkout argouml

    When you have checked out the project, the checked out copy of remembers the cvsroot that it belongs to so you don't need to give the "-d :pserver:blabla:/cvs" in subsequent cvs commands.

  2. To check out the project source repository an individual module (if you don't need the entire repository), type:

    cvs -d :pserver:guest@cvs.tigris.org:/cvs checkout [modulename]
    This command should result in a scrolling list of files being checked out and a copy of them is after that stored in the local directory you specified on your machine. Now you are ready to use your file editor to work with individual files and make changes.

If you wish to contribute to the project's CVS repository, you must have "cvs-commit"-rights as a project member with the role Developer.

CVS how-to guide for this site

WinCvs checkout instructions

To use WinCvs to check out your own set of source code files, you must first set up the correct cvs root using the following steps.

  1. Launch WinCvs and select Cvs Admin - Preferences. Enter the CVSroot:

    :pserver:guest@cvs.tigris.org:/cvs
    Click OK.

  2. If this your first cvs checkout, create a folder in Windows Explorer to hold all of your cvs project folders. Then create a subfolder for this project. (You may even want to create separate subfolders for each module if you're working in more than one.)

  3. In WinCvs, select Cvs Admin - Login and enter your cvs password.

  4. Click on the left window in the program and select a folder. Then select Cvs Admin - Checkout Module. Select the project folder you created earlier.

  5. Enter project module name and click OK. You should see a scrolling list of filenames as these are created in your folder(s).

  6. Repeat the module creation process for each additional cvs module you wish to check out.
