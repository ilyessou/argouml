Internationalization teams

The following people are committed to do the internationalization and localization work in the ArgoUML project.

Each group is responsible for all issues regarding a certain language/region/country.

  • (default) English (American English)
    All developers maintain this while working with the code.
    For ArgoUML the default language is American English (en_US). This is to be used in all non-localized texts.
    Notice that javadoc comments and variable names, log messages (through log4j and System.out), and design documentation (cookbook and web site), are not localized but always written in American English (en_US). This means that all developers of ArgoUML must know English.
    Notice also that there is no (en) files. They are not needed since it is the default.
