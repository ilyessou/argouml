Status of the different translations

(Linus Tolkes very personal opinion!)

All internationalisation effort suffer the problem that the internationalisation in the code is not complete. I (Linus Tolke) suggest that all teams make the internationalisation of the code their first priority and their respective languages their second priority.
Language ArgoUML
fr Not complete since localisation of ArgoUML is not complete
de Not up to date
es Not up to date
en_GB Not all files in place
ru Not started
Suggested priority order for each team:

  1. Complete localisation of ArgoUML excluding gef-parts.
  2. Complete localisation of GEF.
  3. Quality assurance, making sure that the terms used match the translations used in books in the domain.
  4. User manual.
