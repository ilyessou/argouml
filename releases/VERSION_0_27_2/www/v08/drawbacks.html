not implemented parts of UML 1.3

ArgoUML uses Novosoft's NSUML-Library as a metamodel, to store the models in. This library is complete UML 1.3, since it is generated directly from the specification, and offers various features. However, ArgoUML doesn't use all these, and doesn't allow the user to design everything which is possible with UML 1.3. Here's an (incomplete) list of these things:
  • association classes cannot be drawn
  • parametrized classes cannot be drawn
  • stereotypes are not properly displayed, but they are there
  • it cannot be chosen what kind of dependency you are drawing (binding, usage...)
  • the order of messages in collaboration diagrams are is not handled
  • there are no sequence diagrams yet
  • handling multiple top-level models is not possible
    • All these drawbacks will hopefully be handled in one of the following versions of ArgoUML. Please take a look at our developers mailing list for more information.
