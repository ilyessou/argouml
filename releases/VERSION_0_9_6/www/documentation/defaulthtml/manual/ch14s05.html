14.5. ArgoUML Diagrams

ArgoUML supports seven types of diagrams: i) use case; ii) class; iii) sequence; iv) state; v) collaboration; vi) activity; and vii) deployment. They are created using the Create Diagram drop down menu (see Section 12.6).

Note

State and activity diagrams are associated with a particular class, and can only be created when a class has been selected. In the current version of ArgoUML they are not grayed out even when a class is not selected. However under these circumstances they will have no effect.

14.5.1. Diagram Details Tabs

The details tabs that are active for diagrams are described in Table 14.10. Details tabs for diagram.

Table 14.10. Details tabs for diagram

TabMeaning
ToDoItem

Standard tab.

Properties

See Table 14.11. Property tool bar for diagram and Table 14.12. Property fields for diagram below.

Source

Holds a reference to the ArgoUML implementation class for that diagram.

Note

The source tab content can be changed, but this has no effect.

14.5.2. Diagram Properties

The property tab tool bar for diagrams is described in Table 14.11. Property tool bar for diagram.

Table 14.11. Property tool bar for diagram

ButtonIconMeaning
Go back

Navigate back to the previous diagram. Always grayed out, since there is no way to navigate to a diagram (and hence no way to navigate back).

Go forward

Navigate forward to the next diagram. Always grayed out, since there is no way to navigate to a diagram (and hence no way to navigate forward).

The fields on the property tab for diagrams are described in Table 14.12. Property fields for diagram.

Table 14.12. Property fields for diagram

FieldMeaning
Name

The name of the diagram. There are no conventions for naming diagrams. By default, ArgoUML uses the (space separated) diagram name and a sequence number, thus use case diagram 1.
