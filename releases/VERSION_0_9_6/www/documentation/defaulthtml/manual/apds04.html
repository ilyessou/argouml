D.4. Actions, Priorities and Questions

D.4.1. Actions and priorities

Here's my first call for what needs to be done in priority order. From the comments made over the last few days I think the first 5 items won't take very long, meaning effort can concentrate on the main stuff.

  1. Get buy-in for the approach

  2. Agree document structure (broadly)

  3. Choose a suitable example to run throughout.

  4. Break into several files (XML entities) to make the manual more manageable.

  5. Identify all existing sources of material to be reused

  6. Get writing! I suggest the priorities here are:

    1. User Manual sections relating to ArgoUML diagrams and artifacts (assume the reader knows UML already, and allows a quick advance by pulling together a lot of existing material).

    2. User Manual examples

    3. User Manual sections relating to additional ArgoUML cognitive design features.

    4. User Manual sections relating to UML (for readers who don't know UML).

    5. Completion of Reference Manual material.

  7. Create an index.
