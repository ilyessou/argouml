21.2. Node

A node is a run-time physical object on which components may be deployed. In the UML metamodel it is a sub-class of Classifier.

A node is represented on a class diagram as a 3-D box, labeled with its name.

21.2.1. Node Details Tabs

The details tabs that are active for nodes are described in Table 21.1. Details tabs for node.

Table 21.1. Details tabs for node

TabMeaning
ToDoItem

Standard tab.

Properties

See Table 21.2. Property tool bar for node and Table 21.3. Property fields for node below.

Documentation

Standard tab.

Style

Standard tab. The Bounds: field defines the bounding box for the node on the diagram.

Warning

There are some bugs remaining in entering data in the Bounds: field. Take care when typing that you do get the values you want in place.

Altering the Shadow entry has no effect.

Source

Standard tab, but with no contents.

Caution

A node should not generate any code, so having this tab active is probably a mistake.

Constraints

Standard tab. There are no standard constraints defined for Node within the UML metamodel.

Tagged Values

Standard tab. In the UML metamodel, Node has the following standard tagged values defined.

  • persistence (from the supernode, Classifier). Values transitory, indicating state is destroyed when an instance is destroyed or persistent, marking state is preserved when an instance is destroyed.

  • semantics (from the supernode, Classifier). The value is a specification of the semantics of the node.

  • derived (from the supernode, ModelElement). Values true, meaning the node is redundant—it can be formally derived from other elements, or false meaning it cannot.

    Note

    Derived nodes still have their value in analysis to introduce useful names or concepts, and in design to avoid re-computation.

Note

The UML Element metanode from which all other artifacts are derived includes the tagged element documentation which is handled by the documentation tab under ArgoUML

The fields on the property tab for nodes are described in Table 21.3. Property fields for node.

Table 21.3. Property fields for node

FieldMeaning
Name

Text box. The name of the node. The name of a node has a leading capital letter, with words separated by “bumpy caps”.

Note

The ArgoUML critics will complain about node names that do not have an initial capital.

Extends

Text box. Lists any node that generalizes this node.

Caution

ArgoUML is incorrect in using the designation “Extends” for this field. It should correctly be called “Generalizations”.

Button 1 click takes you to the selected entry, button 2 gives a pop up menu with five entries.

  • Open. Navigate to the selected generalization.

    Caution

    In the current release of ArgoUML generalizations of nodes are not implemented, and this button is always grayed out.

  • Add. Add a new generalization.

    Caution

    In the current release of ArgoUML generalizations of nodes are not implemented, and this button is always grayed out.

  • Delete. Removes the link to the selected generalization.

    Caution

    In the current release of ArgoUML generalizations of nodes are not implemented, and this button is always grayed out.

  • Move Up. Only available where there are two or more generalizations, and the generalization selected is not at the top. It is moved one up the list.

    Caution

    In the current release of ArgoUML generalizations of nodes are not implemented, and this button is always grayed out.

  • Move Down. Only available where there are two or more generalizations listed, and the generalization selected is not at the bottom. It is moved one down the list.

    Caution

    In the current release of ArgoUML generalizations of nodes are not implemented, and this button is always grayed out.

Modifiers

Check box, with entries abstract, final and root.

  • abstract is used to declare that this node cannot be instantiated, but must always be specialized. The name of an abstract node is displayed in italics on the diagram.

  • final indicates that this node cannot be further specialized.

  • root indicates the ndoe can have no generalization.

Namespace

Text box. Records the namespace for the node. This is the package hierarchy.

Button 1 click on the entry will take you to the package defining this namespace (or the model for the top level namespace).

Derived

Text area. Lists any derived nodes, i.e those for which this node is a generalization.

Caution

ArgoUML is incorrect in using the designation “Derived” for this field. It should correctly be called “Specializations”.

Button 1 click takes you to the selected entry, button 2 gives a pop up menu with one entry.

  • Open. Navigate to the selected specialization.

    Caution

    In the current release of ArgoUML generalizations of nodes are not implemented, and this button is always grayed out.

Components

Text box. Lists any components (see Section 21.4) designed to be deployed on this type of node.

Button 1 click takes you to the selected entry, button 2 gives a pop up menu with three entries.

  • Open. Navigate to the selected component.

  • Move Up. Only available where there are two or more components, and the component selected is not at the top. It is moved one up the list.

  • Move Down. Only available where there are two or more components listed, and the component selected is not at the bottom. It is moved one down the list.
