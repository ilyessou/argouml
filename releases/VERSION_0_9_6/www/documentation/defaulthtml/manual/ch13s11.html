13.11. Modularity

Critics concerning modular development in ArgoUML.

The current version of ArgoUML has two critics in this category.

13.11.1. Aggregate Role in N-way Association (First Version)

Suggestion that aggregation may not be used for associations with more than two ends.

Warning

Whilst it is possible to create N-way associations in the current release of ArgoUML, it is not supported graphically, and of little use. It is a known limitation of the current release that if you do create such associations, they will always trigger this critic, even if there is no aggregation on any end.

13.11.2. Add Elements to Package <package>

Suggestion that the specified package has no content. Good design suggests packages are created to put things in.

Note

This will always trigger when you first create a package, since you cannot create one that is not empty!
