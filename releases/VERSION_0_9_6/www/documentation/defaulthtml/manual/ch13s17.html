13.17. Containment

Critics concerning containment in ArgoUML, that is where one artifact forms a component part of another.

The current version of ArgoUML has six critics in this category.

13.17.1. Remove Circular Composition

Suggestion that there is a series of composition relationships (associations with black diagrams) that form a cycle, which is not permitted.

13.17.2. Duplicate Parameter Name

Suggestion that a parameter list to an operation or event has two or more parameters with the same name, which is not permitted.

13.17.3. Multiple Aggregate Roles

Suggestion that more than one end of an association is aggregate or composite. At most one end should have multiplicity of any sort.

13.17.4. Aggregate Role in N-way Association (Second Version)

Suggestion that more than one end of an association is aggregate or composite. At most one end should have multiplicity of any sort.

Caution

This appears to duplicate the critic Multiple Aggregate Roles within this design category (see Section 13.17.3 and the critic of the same name under the modularity design issue (see Section 13.11.1).

13.17.5. Wrap DataType

This critic is discussed under an earlier design issues category (see Section 13.3.1).

13.17.6. Import Parameter Type into Class

Suggestion that the type of a parameter used within an operation of a class is not visible.

Warning

It is not clear how this can ever be caused within ArgoUML. In any case this critic can never trigger in the current implementation of ArgoUML.
