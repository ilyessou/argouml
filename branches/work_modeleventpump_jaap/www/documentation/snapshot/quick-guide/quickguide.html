Preface

1 April 2001

In the last decade of the last millenium, object-oriented (OO) technology finally completed it's transformation from a laboratory curiousity in the 1960s to become the mainstream software development paradigm. It has been a long, tough journey - that nearly ended in obscurity - primarily because OO technology required a major shift in the thought processes of systems designers, developers and others involved in the software development lifecycle.

It wasn't until the introduction of OO-hybrid languages such as C++, Classic Ada and Object Pascal in the eighties that the mainstream was introduced to and, given the opportunity to experiment with OO technology. These languages combined support for OO programming with support for traditional procedural programming. The lack of support of procedural programming in pure OO languages had hindered the adoption of OO technology for many years. The journey that had begun with the introduction of objects in Simula in the 1960s was now complete. OO technology was now accessible to the mainstream...well, OO programming was at least.

Simulation is the basic premise that underpins OO technology. An OO system is designed and implemented essentially as a simulation of the real world using software artifacts. This premise is as powerful as it is simple. By designing and buidling software systems in this manner, the same language and ideas can be used in the analysis, design and implementation of OO systems. This allowed a system to be designed and tested (or more correctly allowed a system to be simulated) without having to actually build the system first. This feature coupled with the ability to design systems at a very high level empowered experienced OO practitioners to design and successfully implement more complex systems than had previously been possible.

The availabilty of various OO methodologies and the eventual adoption of the Unified Modelling Language (UML) as the standard language for communicating OO concepts continued the advance of OO technology into the mainstream. The popularity of the object-based Visual Basic language and the parallel development and meteoric adoption of the fully object-orinted Java programming language was the final catlayst that fuelled the final stages of the rapid adoption of OO technologies by the mainstream.

ArgoUML was conceived as a tool and environment for use in the analysis and design of object-oriented software systems. In this sense it is similar to many of the commercial CASE tools that are sold as tools for modelling software systems. ArgoUML has a number of very important distictions from many of these tools:

  • ArgoUML includes a number of features that support the cognitive needs object-oriented software designers and architects.

  • ArgoUML supports open standards extensively - UML, XMI, SVG, OCL and others. In this respect, ArgoUML is still ahead (even almost five years after it's initial introduction) of many commercial tools.

  • ArgoUML is a 100% Pure Java application. This allows ArgoUML to run on all platforms for which a reliable port of the Java2 platform is available.

  • ArgoUML is an open source product. The availability of the source permits ensures that a new generation of software designers and researchers now have a proven framework from which they can drive the development and evolution of CASE tools technologies.

In creating the ArgoUML development tool and environment, Jason Robbins and the rest of his research team at UCL leveraged the benefits of UML as the most prevalent OO modelling language and Java as one of the most productive OO development platforms to produce a solid tool for OO systems design and, further a testbed for the evolution of OO CASE tools development and research.

1.1. Welcome to ArgoUML

Welcome to ArgoUML from the ArgoUML project team at Tigris. ArgoUML is a powerful yet easy-to-use interactive, graphical software design environment that supports the design, development and documentation of object-oriented software applications. It leverages the Write Once, Run Anywhere promise of the Java2 platform to allow you to design great object-oriented software in your preferred Operating System environment.

If you are familiar with a family of software applications called Computer Aided Software Engineering (CASE) tools then you should find ArgoUML instantly familiar. It is an interactive, graphical modelling CASE tool that is easy to install, learn, use and customise for your own unique design environment. It is built on an extendible foundation of open standards (or industry standards) to ensure interoperability with the rest of the industry. You can share designs created with ArgoUML and collaborate with colleagues using other tools easily because of it's support for open standards.

The typical users of ArgoUML include software designers & architects, software developers, business analysts, systems analysts and other professionals involved in the analysis, design and development of software applications. It has a number of features (some unique) that enables it to better support these users in the design of software systems:

  • Uses the open standard Unified Modelling Language (UML) as the language for documenting and communicating object-oriented designs

  • Supports open standards including XML Metadata Interchange format (XMI), Scalable Vector Graphics format (SVG) and Portable Graphics Markup Language natively and as formats for the communicating design information with external systems

  • Is a 100% Java application to ensure that it is available for use on all the major OS platforms plus any other OS an implementation of the Java2 platform exists

  • Is an Open Source product that enables it's users to extend or customize it. Many users also eventually become contributors to the future development of ArgoUML itself. On particularly important group of users are researchers and research groups who often endow ArgoUML with state-of-the-art features ahead of other CASE tools

  • Incorporates a number of specific cognitive features to better support the design process:

    • reflection-in-action

    • opportunistic design

    • comprehension and problem solving

ArgoUML was originally a product of a research programme at UCL led by Jason E. Robbins.

ArgoUML version 0.8.x or earlier

This guide is designed for the 0.9.x versions of ArgoUML. Many of the comments may also be applicable to earlier versions of the software but that has not been tested.

1.2. About this guide...

This guide - the ArgoUML QuickStart Guide - is provided to help you get up and running as quickly as possible with ArgoUML. It provides step-by-step instructions for obtaining and installing ArgoUML, performing common tasks, and outlines your options for contacting ArgoUML project team should you need further support or to contribute to ArgoUML's development.

1.2.1. In this guide, you will find...

The remainder of this chapter includes information about documentation conventions and what you need to know to be able to use ArgoUML effectively. Here's what you can expect to find in the remaining chapters in this guide:

  • Download and Installation instructions

  • Basic tutorial/introduction to ArgoUML - step-by-step procedures on essentials

  • Support option - mailing list, resources, contributions, source code/CVS etc..

1.2.2. In this guide, you will not find...

This guide is not a complete user or reference manual for ArgoUML. The ArgoUML project website has information about the status of other ArgoUML documentation including a user manual.

This guide does not attempt to teach you about UML, XMI, SVG or any other such standards that ArgoUML implements or supports. Neither does it attempt to teach you about the Java platform or your Operating System platform.

1.2.3. What you should already know

This guide assumes that you know how to use your computer. It also assumes that you have some knowledge of object-oriented software design given your interest in ArgoUML. You should be familiar with the process of installing software applications on your computer although full installation instructions are given for ArgoUML.

Once ArgoUML is installed, it is assumed that you know how to point, click, and drag with a mouse or another pointing device and that you can use a keyboard.

1.2.4. Documentation conventions

Before reading this guide, please take a moment to ensure that you understand the conventions followed in the guide. This will help you to use the guide more effectively.

1.2.4.1. Conventions related to the use of the mouse

When you see this...Do this...
Click Press the primary mouse button (usually the left mouse button).
Double-click Press the primary mouse button quickly twice.
Right-click Press the secondary mouse button (usually the right mouse button).
Files->New Click File on the top-level menu bar and, click the New option in the displayed 'File' menu.
Create->Diagrams->ClassDiagram Click Create on the top-level menu bar, click Diagrams in the 'Create' menu that appears, click the ClassDiagram option in the 'Diagrams' submenu that appears.
Select text Click and drag to highlight text.

1.2.4.2. Conventions related to the use of the keyboard

When you see this...Do this...
Enter Press the Enter key on your keyboard.
Control+Shift Press the Control key and the Shift key at the same time.
Control-C Press the Control key and while keeping it pressed, press the C key at the same time.

2.1. Can your computer run ArgoUML?

Before you can install and use ArgoUML you must ensure that your computer's configuration meets the minimum system requirements for running ArgoUML. To ensure that your computer can run ArgoUML properly we will check the following:

  • Your computer's hardware specification

  • The version of the Java Platform installed on your computer (if any)

2.1.1. Check your computer's hardware specification

ArgoUML is a complex application with an interactive, graphics editing user interface. The minimum recommended configuration for a computer running ArgoUML is shown below. Please check and ensure that your computer meets the following minimum requirements before installing or using ArgoUML:

  • Microsoft Windows 95/98/Me/NT/2000, Linux or other OS that supports Java2

  • 200MHz Intel Pentium, AMD K6 or similar processor

  • 64MB (128MB for Windows 2000) RAM

  • 10MB of free disk storage space on your hard disk

  • SVGA video output and monitor

  • Mouse (or other pointing device) and a Keyboard

2.1.2. Check the version of Java on your computer

ArgoUML is a 100% Pure Java application. Your computer must have a version of the Java 2 platform JDK installed to use ArgoUML. More specifically, the Java 2 JDK v1.2 (i.e. JDK1.2 or later) is recommended for use with ArgoUML. Although ArgoUML has not been tested an all Java 2 platforms, it should run on all Java 2 installations regardless of the underlying OS platform.

Note

Official Java 2 JDKs are available to download from Sun at http://java.sun.com/jdk/.

2.2. Installation options

Now that you have a computer that can install and run ArgoUML effectively, you need to decide which of the installation options available is appropriate for you. You can choose from the following options:

  • Java Web Start option - this option is provided for first-time or occasional users. It is the easiest and quickest way to start using ArgoUML

  • Binary distribution option - this option is provided for regular users. You should also choose this option if you are having problems with the Java Web Start option.

  • Source or CVS distribution option - this option is provided for ArgoUML developers and contributors. You should choose this option if you are having problems with the other options.

2.2.2. Binary distribution option

This is the most common installation option for regular ArgoUML users. It involves obtaining a copy of the binary distribution archive from the ArgoUML project website and then preparing your computer to be able to run ArgoUML using the contents of the binary distribution. You should use this option if you are either unable or unwilling to run the Java Web Start version or, your computer is not directly connected to the internet or, you already have the binary distribution.

2.2.2.1. Obtain the ArgoUML binary distribution

If you haven't already done so, please download a copy of the binary distribution from the ArgoUML homepage. If the computer on which ArgoUML will be installed is not connected to the internet, download the file using another computer and copy the binary distribution file to the target computer.

2.2.2.2. Create an ArgoUML installation directory

Create a directory on your computer for the ArgoUML application. This directory will thereafter be referred to as the ArgoUML installation directory or just as ARGOUML_HOME. Sample commands for creating an ArgoUML installation directory on computers running Windows and Linux are shown below. You might want to choose a different location for ArgoUML on your computer.

Example 2.1. Creating an installation directory

Sample commands for Windows 

mkdir C:\OO-Tools\ArgoUML

Sample commands for Linux and other Unixes 

md /usr/bin/ArgoUML

Example 2.2. Creating an installation directory

Sample commands for Windows 

mkdir C:\OO-Tools\ArgoUML

Sample commands for Linux and other Unixes 

md /usr/bin/ArgoUML

2.2.2.3. Extract the ArgoUML files to the installation directory

You will now change the current directory to the ArgoUML installation directory you have just created and extract the files contained in the binary distribution to the directory. The general form of the command to extract the files is:

jar xvf ArgoUML-archive-filename

As an example, let's say you have an ArgoUML binary distribution archive named ArgoUML-0.9.3.zip that it is stored in a directory called C:\Distributions or ~/distributions on a Windows or Linux computer respectively. To extract the ArgoUML files from the archive, you would type the following commands on the Windows and Linux/Unix platforms.

Example 2.3. Extracting ArgoUML files from binary distribution

chdir C:\OO-Tools\ArgoUML                                       
jar xvf C:\Distributions\ArgoUML-0.9.3.zip

cd /usr/bin/ArgoUML                                             
jar xvf ~/distributions/ArgoUML-0.9.3.zip

You should now have the ArgoUML application's files copied to the installation directory on your system.

2.2.3. Source or CVS distribution option

This is the preferred installation option for ArgoUML developers, contributors and advanced users. It involves obtaining the ArgoUML source code as a distribution from the ArgoUML project website or from CVS, building ArgoUML on your computer, and then preparing your computer to run ArgoUML.

You should use this option if you want to keep up with the latest developments, you are a developer or contributor on the ArgoUML project or if you have problems using the other installation options. Please note however that the devlopment copy of ArgoUML is quite often unstable and is not advised for production use.

2.2.3.1. Obtain the ArgoUML source code

You can obtain the ArgoUML source code via one of two methods:

  • Source archive - this is a snapshot of the entire ArgoUML source code that is updated for each release (and maybe at other times too). This is the easiest way to get the ArgoUML source code but the contents may not be the latest available ArgoUML source code. You will need to obtain two archive files, one for the ArgoUML source code and one for the add-ons that ArgoUML depends on.

  • CVS checkout - in this case you will extract the latest copy of the ArgoUML source code from the public CVS database that the ArgoUML project uses to track the development of ArgoUML itself. This is really only recommended for developers and other contributors as the latest source code is also the least stable version of ArgoUML. When you use this method, what you get is always a snapshot of ArgoUML's source code as it is being developed.

If you haven't already done so, please please obtain a copy of the ArgoUML source code here. If the computer on which ArgoUML will be installed is not connected to the internet, download the file(s) using another computer and copy the file(s) to the target computer.

2.2.3.2. Extract the ArgoUML source code (source archive method only)

If you have chosen to obtain the ArgoUML source distribution archive(s), you need to extract the files from the archive(s) to a directory on your computer before you can build your copy ArgoUML.

Firstly, create a directory on your computer for the ArgoUML source code. This directory will thereafter be referred to as the ArgoUML source directory or just as ARGOSOURCE_HOME. Sample commands for creating an ArgoUML source directory on computers running Windows and Linux are shown below. You might want to choose a different location for ArgoUML on your computer.

Example 2.5. Creating a source directory

Sample commands for Windows 

mkdir C:\OO-Tools\ArgoUML-source

Sample commands for Linux and other Unixes 

md /usr/bin/ArgoUML-source

Next, change the current directory to the ArgoUML source directory you have just created and extract the files contained in the source distribution archive(s) to the directory. The general form of the command to extract the files is:

jar xvf ArgoUML-archive-filename

As an example, given the following assumption on a Windows computer (with Linux/Unix-specific details in brackets):

  • an ArgoUML source archive named ArgoUML-0.9.3-src.zip (ArgoUML-0.9.3-src.tar.gz on Linux/Unix)

  • an ArgoUML add-ons source archive named ArgoUML-0.9.3-libs.zip (ArgoUML-0.9.3-libs.tar.gz on Linux/Unix)

  • both source archives are stored in the directory C:\Distributions (~/distributions on Linux/Unix)

To extract the files from both archives to the ArgoUML source directory, you would type the following commands on the Windows and Linux/Unix platforms.

Example 2.6. Extracting ArgoUML files from source distribution

Sample commands for Windows 

chdir C:\OO-Tools\ArgoUML-source
jar xvf C:\Distributions\ArgoUML-0.9.3-src.zip
jar xvf C:\Distributions\ArgoUML-0.9.3-libs.zip

Sample commands for Linux and other Unixes 

cd /usr/bin/ArgoUML-source
jar xvf ~/distributions/ArgoUML-0.9.3-src.tar.gz
jar xvf ~/distributions/ArgoUML-0.9.3-libs.tar.gz

Note

For the above command to work, the bin sub-directory of the JDK directory tree must be added to your PATH environment variable. You might also be able to use a compatible de-archiver such as winzip or gzip to extract the files.

You should now have the ArgoUML application's source files copied to the ArgoUML source directory on your system.
