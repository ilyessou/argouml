ArgoUML in Google's Summer Of Code 2006

The ArgoUML project is, for the first time during 2006, attempting to take part in Google's Summer of Code.

We are primarily looking for students interested in Software Engineering or Systems Engineering, tools for Software Engineering, and communication within a Software Engineering project using UML. Java and UML are the natural prerequisites.

This page include a list of projects that are especially interesting for 2006. Don't let this limit you. If you have other projects in mind, let us know and we will evaluate your ideas with the rest of the projects. If you want you can also look at our old Job Jar for inspiration.

ArgoUML performance

ArgoUML is sometimes slow and uses a lot of memory. In the big picture this is cumbersome for the users that need to sit and wait hours for batch operations and in the small picture it is irritating and creates a lot of user misstakes when popup menus and drag and drop operations take seconds to update. It is suspected that some simple measures can be take to improve this.

This project will include:

  • Running ArgoUML in some performance measuring tool.
  • Examining the result, finding areas where improvements could be done.
  • Make the improvements.
  • Restart.

Unless knowledge of and access to some performance measuring tool for java, this will need to be acquired.

ArgoPrint improvements

ArgoPrint is a tool to extract the information from the UML model and present it in written form as a report.

Two things needs to be done to it.

  • Improve it to make it usable.
  • Improve the template editing capabilities.

Linus Tolke will be the mentor for this.

GEF improvements

GEF is the Graph Editing Framework used by ArgoUML for rendering UML diagrams.

Four things needs to be done to it.

  • Complete undo facility
  • Implement graph layout algorithms
  • Add Eclipse compatibility
  • Implement demonstration projects

Bob Tarling will be the mentor for this.
