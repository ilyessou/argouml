Argo/UML JobJar


Take on a Task
A lot of work has gone into developing Argo/UML, but much more still remains to be done.

Here is a list of pending tasks that you might be interested in doing. Each of these tasks would be a personally satisfying contribution toward making the Argo/UML vision a reality. Please contact us to discuss tasks that you are interested in.

To show our appreciation, we will add you name and link to your home page to the Argo/UML credits in recognition of your contribution. We are also planning on giving out project T-shirts at some future time.

Special Note to Students
Each task could be made into an undergraduate term project or part of a masters degree project. Working on a free, open source software project such as Argo/UML can make your school project much more impressive, satisfying, and clarify potential benefits. Why start with nothing if you could build on the work of over a dozen other students? Why end your term project with just a grade when you could make a visible difference? The point of a term project is to get experience working on a "real" project. Using Argo/UML will give you experience with over 60,000 lines of Java code that implements a very real application.


Tasks

  • Use Argo/UML
    • Try out Argo/UML and make notes of things that are buggy or awkward.
    • Try to use it for a small design project. What is useful about the tool? What is needed?
    • Watch others use Argo/UML and report on how they use it.
  • UML Diagrams
    • Implement a graph layout algorithm that takes into account the conventions of UML diagrams.
    • Implement arc routing that users find flexible and convient.
    • Implement diagram types that are part of UML but not supported by Argo/UML yet.
    • Implement printing of wall-sized diagram posters or design booklets.
    • Implement zooming.
    • For more ideas on diagram editing enhacements, see the GEF Web Site.
  • UML Meta-Model
    • Add an JDBC interface to the UML meta-model so that models can be stored in databases.
    • Add the ability to save the UML meta-model to a file using Java's ObjectSerialzation library.
    • Add an external file format for compatability with other tools, oe enhance Argo/UML's XMI support.
    • Generate an HTML reference web site for a UML model, much like JavaDoc does for Java code.
    • Recode parts of the meta-model to remove some current simplifying assumptions.
  • Code Generation and Reverse Engineering
    • Implement code generation for an additional programming language.
    • Implement support for generating Java code that follows JavaBeans conventions.
    • Implement an extensible and customizable library for code generation in many languages.
    • Implement code generation for UML state machine models.
    • Generate assert() statements to check OCL constraints on a UML model at run-time.
    • Integrate a Java parser into Argo/UML to accept fragments of code entered in diagrams.
    • Use a Java parser to read code from files and construct a model.
  • User Interface
    • Implement more table views of UML packages, classes, state machines, etc.
    • Improve any aspect of Argo's user interface and show that it is better.
    • Implement Undo.
    • Add pop-up menus to several parts of Argo's interface.
    • Implement a vertically split navigator pane: lower half to show details of the upper selection.
  • Documentation
    • Write a users guide for Argo/UML, focus on a specific task or audience.
    • Design and outline a maintainable user's guide for others to fill in over time.
    • Write a developers guide to help new people contribute to the Argo/UML project.
  • Object Constraint Language
    • Integrate the OCL parser into Argo/UML.
    • Enhance Argo/UML's OCL evaluator.
  • Cognitive Support for Design
    • Implement some critics or checklist items that give advice you think is useful.
    • Implement critics or checklists based on ideas from books or company policies.
    • Implement wizards to help fix problems.
  • Performance
    • Measure what parts of Argo/UML take time or space.
    • Enhance performace in some measurable way.
