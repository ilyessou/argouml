The Design of ArgoUML

This is an attempt to use html to develop a design that we will have as goal for ArgoUML.

This is work in progress. I havn't even added the components yet. Please let me know what you think about the ideas...

These are the components (how they were defined by Linus in the Cookbook autumn 2002 is the initial attempt):

  • Model
  • Critics
  • Diagrams
  • Property panels
  • Reverse Engineering
  • Code Generation
  • Java
  • Other languages
  • GUI Framework
  • Application
  • Help System
  • Internationalization
  • Logging
  • JRE with utils
  • To do items
  • Explorer
  • Module loader
  • OCL

How we work together on this

Feedback and status

This is a work in progress for the design of ArgoUML of the future. When things are finished and have settled down in the design you can start augmenting the design with the migration path from the old to the new design. When that migration path is completed you can start implementing the new desing in the code.

Monitor the cvs mailing list for changes to any of these files if you want to follow the development.

If you don't agree on what someone has written, change it, unless this is the basis for a lot of other work in the design.

State of mind for this work

  • If there are omissions in any document, amend it!
  • If you don't have the time, knowledge, or inclination to finish the sentence, the paragraph, or the chapter just enter a TODO:-stating what is not in place. You can always come back later or someone else can do it.
  • If you don't have the possibility to edit the files, ask someone that can to do it for you (don't ask on the mailing lists).

Ground rules

  • Everything is contained under www/design.
  • Use the simplest possible html!
  • Type setting things are banned i.e. things like fonts or styles. (This kind of thing are added by the Tigris site so it will probably be a mess anyway.)
  • Store pictures in /design/pictures/.
  • Commit often.
  • No watches. Keep text files short (never over 50KB).
