ArgoUML History

This is the ArgoUML History depicted as a timeline. Currently it is very incomplete because the initial author (Linus Tolke) don't know that much about the ancient history of ArgoUML. Hopefully some of the persons that has been around for long (Jason, Marko, Toby...) can fill in the blanks and questions marks and answer the questions.
WhenWhat
?? ArgoUML was started as part of the Ph.D. research of Jason Robbins.
1996 June First reference to the name Argo name in the paper: "Extending Design Environments to Software Architecture Design" by Jason E. Robbins, David M. Hilbert, and David F. Redmiles. Published in Knowledge-Based Software Engineering 1996.
July 1998 The first preliminary versions of Argo/UML were distributed from the university web site (www.ics.uci.edu).
?? The Argo family of domain oriented design environments was created and ArgoUML was made part of it. This family of tools was developed for over three years. Other tools in the family support C2-style software architectures and requirements engineering. All tools in the Argo family allow for construction of design artifacts and provide decision-support features to help designers make better designs. Domain-indepentent features of the Argo infrastructure include: critics, user models, the "to do" list metaphor, checklists, multiple perspectives.

What happened to this family of tools then? Is ArgoUML the only remains or are there other tools?

In 1998 ArgoUML matured into a usable CASE tool with special features to help designers with the cognitve challenges of object-oriented design.
1999 February ArgoUML was made into an Open Source project.

??Curt Arnold created the new property panels

?? Toby Baier introduced NSUML as a substitute for ArgoUMLs own metamodel. This work was part of his master thesis.
