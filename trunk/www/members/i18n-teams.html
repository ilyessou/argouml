Internationalization teams

The following people are committed to do the internationalization and localization work in the ArgoUML project.

Each group is responsible for all issues regarding a certain language/region/country.

  • (default) English (American English)
    All developers maintain this while working with the code.
    For ArgoUML the default language is American English (en_US). This is to be used in all non-localized texts, comments and documentation.
    Notice that javadoc comments and variable names, log messages (through log4j and System.out), and design documentation (cookbook and web site), are not localized at this point but always written in American English (en_US).
  • (en) English
    All developers maintain this while working with the code.

Status of the different translations

(Linus Tolkes very personal opinion!)

All internationalization effort suffer the problem that the internationalization in the code is not complete. I (Linus Tolke) suggest that all teams make the internationalization of the code their first priority and their respective languages their second priority. Jean-Hugues de Raigniac coordinates the work with completing the internationalization throughout the code.

Report for each bundle on 08/02/2002 : 20020802.tar.gz (23ko).

Suggested priority order for each team:
  1. Complete localization of ArgoUML excluding gef-parts.
  2. Complete localization of GEF.
  3. Quality assurance, making sure that the terms used match the translations used in books in the domain.
  4. User manual.
