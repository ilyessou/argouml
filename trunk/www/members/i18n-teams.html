Internationalization teams

The following people are committed to do the internationalization and localization work in the ArgoUML project.

Each group is responsible for all issues regarding a certain language/region/country.

  • (default) English (American English)
    All developers maintain this while working with the code.
    For ArgoUML the default language is American English (en_US). This is to be used in all non-localized texts.
    Notice that javadoc comments and variable names, log messages (through log4j and System.out), and design documentation (cookbook and web site), are not localized but always written in American English (en_US). This means that all developers of ArgoUML must know English.

Comments on the work of the teams

All internationalization effort suffer the problem that the internationalization in the code is not complete. I (Linus Tolke) suggest that all teams make the internationalization of the code their first priority and their respective languages their second priority. Jean-Hugues de Raigniac coordinates the work with completing the internationalization throughout the code.

On the other hand it is not really that simple because in order to find the places where the code is not complete you have to work with an ArgoUML where all the property files has been translated so you have a lot of translation work in order to get there.

Suggested work order for each team:

  1. Complete localization of ArgoUML excluding GEF-parts (priority 2). When a problem with the code is encountered, report it in Issuezilla and help in solving it (priority 1).
  2. Complete localization of GEF (work with the GEF team).
  3. Quality assurance, making sure that the terms used match the translations used in books in the domain.
  4. Following up on the new development of ArgoUML. The checkstyle tools reports when a tag is missing from a specific language i.e. when there is a missmatch between the non-localized file and any localized file. Check and fix that regularly for your language!
  5. User manual.
