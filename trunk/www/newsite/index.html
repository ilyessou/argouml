Welcome to ArgoUML!

The aims of ArgoUML [10 Mar 2001]:

The goal of the ArgoUML project is to build an object oriented design tool that is:

  • A joy to use (we really mean it),
  • Actually helpful to designers when they are making design decisions,
  • Completely open source Java,
  • Leading edge (supports UML 1.3),
  • Modular and extensible,
  • Integrated with the web and other Tigris tools.
For a more comprehensive list of features, look at the ArgoUML features page.

The ArgoUML website itself, strives to give you a resource for everything which has something to do with the Unified Modeling Language. We therefor welcome your contributions. Whether you know a book, have data about a conference, useful UML links to sites, mailinglists or user groups, drop us a line at [mailto://umlresources@argouml.tigris.org] and we will include it in our UML resources section.

ArgoUML requirements and versions [10 Mar 2001]:

Currently there are two versions of ArgoUML: the slightly older version 0.8.1a andthe newer 0.9.x branch. We recommend that you use ArgoUML 0.8.1a when youintend to use ArgoUML in production environments and ArgoUML 0.9.x when you want toexperiment with ArgoUML and simply want to have a look at its features.ArgoUML 0.9.x has many more features than ArgoUML 0.8.1.a but we will notguarantee that you will be able to load models created with this branch infuture ArgoUML versions. ArgoUML 0.8.1.a models will however continue tobesupported in future versions. The reason for that is that the underlyingmodelto save and load data is currently under heavy change and improvement- wewill write converters from the last stable version (that is ArgoUML0.8.1.a) to the newer systemonce it had been finally established.
